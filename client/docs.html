Pokédex API

Browse API Docs

Overview

All data endpoints return application/json by default. Every response includes Content-Type and Content-Length headers. All GET endpoints also support HEAD (same headers, no body).

Endpoints

/api/pokemon

GET HEAD

Returns all pokemon. Supports filtering via query parameters.

Query ParamTypeRequiredDescription
typestringoptionalFilter to pokemon that include this type (case-insensitive). e.g. Fire
weaknessstringoptionalFilter to pokemon that have this weakness (case-insensitive). e.g. Water
namestringoptionalFilter to pokemon whose name contains this substring (case-insensitive). e.g. saur
limitnumberoptionalMaximum number of results to return.

Returns: { count: number, pokemon: Pokemon[] }

GET /api/pokemon?type=Fire&limit=3

{
  "count": 3,
  "pokemon": [
    { "id": 4, "num": "004", "name": "Charmander", "type": ["Fire"], ... },
    ...
  ]
}

/api/pokemon/:id

GET HEAD

Returns a single pokemon by numeric id or zero-padded num string.

URL ParamTypeRequiredDescription
:idnumber or stringrequiredNumeric id (e.g. 25) or num string (e.g. 025)

Returns: A single Pokemon object. 404 if not found.

GET /api/pokemon/25

{ "id": 25, "num": "025", "name": "Pikachu", "type": ["Electric"], ... }

/api/types

GET HEAD

Returns all unique pokemon types present in the dataset, sorted alphabetically.

Returns: { types: string[] }

GET /api/types

{ "types": ["Bug", "Dragon", "Electric", "Fighting", "Fire", ...] }

/api/weaknesses

GET HEAD

Returns all unique pokemon weaknesses present in the dataset, sorted alphabetically.

Returns: { weaknesses: string[] }

GET /api/weaknesses

{ "weaknesses": ["Bug", "Dark", "Electric", "Fighting", "Fire", ...] }

/api/pokemon

POST

Adds a new pokemon to the in-memory dataset. Data persists until the server restarts.

Accepts application/json or application/x-www-form-urlencoded (parsed based on Content-Type).

For form-urlencoded, type and weaknesses should be comma-separated strings.

Body ParamTypeRequiredDescription
namestringrequiredPokemon name
typestring[]requiredArray of type strings (e.g. ["Fire"])
heightstringoptionalHeight string (e.g. "0.40 m")
weightstringoptionalWeight string (e.g. "6.0 kg")
weaknessesstring[]optionalArray of weakness type strings
imgstringoptionalURL to pokemon image

Returns: 201 with the created Pokemon object. 400 if required fields are missing.

POST /api/pokemon
Content-Type: application/json

{ "name": "Eevee", "type": ["Normal"], "height": "0.30 m", "weight": "6.5 kg" }

→ 201
{ "id": 152, "num": "152", "name": "Eevee", "type": ["Normal"], ... }

/api/pokemon/update

POST

Updates one or more fields on an existing pokemon identified by id.

Accepts application/json or application/x-www-form-urlencoded.

Only fields present in the body are updated — omitted fields are left unchanged.

Body ParamTypeRequiredDescription
idnumberrequiredNumeric id of the pokemon to update
namestringoptionalNew name
typestring[]optionalNew type array
heightstringoptionalNew height
weightstringoptionalNew weight
weaknessesstring[]optionalNew weaknesses array
imgstringoptionalNew image URL

Returns: 204 No Content on success. 400 if id is missing. 404 if no pokemon has that id.

POST /api/pokemon/update
Content-Type: application/json

{ "id": 25, "name": "Raichu" }

→ 204 (no body)

Pokemon Object Shape

{
  "id":           number,    // numeric id (1+)
  "num":          string,    // zero-padded string, e.g. "025"
  "name":         string,
  "img":          string,    // image URL (may be empty for user-added pokemon)
  "type":         string[],  // e.g. ["Electric"]
  "height":       string,    // e.g. "0.40 m"
  "weight":       string,    // e.g. "6.0 kg"
  "weaknesses":   string[],  // e.g. ["Ground"]
  "next_evolution": [        // empty array if final form
    { "num": string, "name": string }
  ],
  "prev_evolution": [        // present on evolved forms only
    { "num": string, "name": string }
  ]
}
